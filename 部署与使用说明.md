## 二、启动方式

### 方式一：一键启动（推荐）

**双击** `C:\PA_Agent\start_pa_agent.bat`

脚本会自动：
1. 切换到项目目录 `C:\PA_Agent`
2. 调用 Python 启动 `run.py`

打开命令行（CMD 或 PowerShell），执行：

```bat
cd /d C:\PA_Agent
python run.py
```

### 方式三：在 WorkBuddy 中启动

直接对 WorkBuddy 说："启动 PA Agent" 或 "运行 C:\PA_Agent\start_pa_agent.bat"。

## 四、如何使用 PA Agent 分析国内期货

### 第 1 步：选择数据源

在主界面顶部工具栏的「数据来源」下拉框中，确认已选中 **「东方财富期货」**。
（启动后默认就是该选项）

### 第 2 步：选择品种和合约（两级选择）

东方财富期货数据源采用**两级选择**：

1. **「品种」下拉框**：选择品种大类（如"RB0 螺纹钢"、"AO0 氧化铝"、"AU0 黄金"），共 27 个品种
2. **「合约」下拉框**：选择该品种的具体合约，自动列出：
   - 第一项始终是**主力连续合约**（如"AO0 主力"）
   - 后续是当前年和明年的各月份合约（如"AO2607"、"AO2608"…）

> 也可以在「合约」框中直接输入任意合约代码（如 `AO2509`、`RB2610`），程序自动识别。

#### 常用期货品种

| 品种 | 中文名 | 品种 | 中文名 |
|------|--------|------|--------|
| RB0 | 螺纹钢 | AU0 | 黄金 |
| HC0 | 热卷 | AG0 | 白银 |
| I0 | 铁矿石 | CU0 | 铜 |
| J0 | 焦炭 | AL0 | 铝 |
| JM0 | 焦煤 | AO0 | 氧化铝 |
| RU0 | 橡胶 | ZN0 | 锌 |
| FU0 | 燃油 | NI0 | 镍 |
| MA0 | 甲醇 | TA0 | PTA |
| PP0 | 聚丙烯 | Y0 | 豆油 |
| P0 | 棕榈油 | M0 | 豆粕 |
| C0 | 玉米 | SR0 | 白糖 |
| CF0 | 棉花 | IF0 | 沪深300股指 |
| IH0 | 上证50股指 | T0 | 十债 |

**代码规则**：字母（1-3个）+ `0` 表示主力连续合约。例如：
- `RB0` → 螺纹钢主力
- `AU0` → 黄金主力
- 输入 `rb` 会自动补全为 `RB0`

### 第 3 步：选择 K 线周期

在「周期」下拉框中选择分析周期。东方财富期货数据源支持：

- **1m / 5m / 15m / 30m** —— 分钟线（日内交易）
- **1h / 4h** —— 小时线（波段交易，4h 由 1h 重采样）
- **1d** —— 日线（中长线趋势）

### 第 4 步：点击「获取数据」

点击界面上的「获取数据」按钮，程序会从东方财富（akshare 期货接口）拉取最近 100 根 K 线（默认）。

K 线图会显示在主界面，包含 OHLC 蜡烛图、EMA20、ATR14 等指标。

### 第 5 步：运行 AI 两阶段分析

1. 确认 K 线图加载完成后，点击「分析」按钮
2. 程序会进行 **两阶段 AI 分析**：
   - **阶段一（Stage 1）**：识别 K 线形态、结构位、趋势上下文
   - **阶段二（Stage 2）**：综合研判，给出交易决策（多空方向、入场位、止损止盈、置信度）
3. 分析结果会显示在「决策」面板，包含：
   - 市场结构判断
   - 关键支撑/阻力位
   - 价格行为信号
   - 交易倾向（保守/平衡/激进）
   - 下单机会提示（限价单/突破单/市价单）

### 第 6 步：查看决策详情

- **决策面板**：查看 AI 给出的具体交易建议
- **决策树可视化**：展示推理过程
- **K 线图叠加**：决策点位会标注在 K 线图上

> 交易时段内（日盘 09:00-11:30、13:30-15:00；夜盘 21:00 后），程序会自动刷新最新价更新当根 K 线。

## 七、常见问题

### Q1：启动时提示 "MT5 initialize() failed"
**A**：这是正常的。本地未安装 MetaTrader 5 终端，不影响东方财富期货数据源使用。该警告可忽略。

### Q2：启动时提示 "tvDatafeed nologin method"
**A**：TradingView 未登录，仅影响 TradingView 数据源。使用东方财富期货时不受影响。

### Q3：获取数据时报错 "未返回期货数据"
**A**：检查品种代码是否正确。必须是主力合约代码（字母+0），如 `RB0`、`AU0`。可输入小写，程序会自动转大写。

### Q4：分钟线只有约 1000 根
**A**：akshare 期货分钟接口返回最近约 1023 根 K 线，这是接口限制。如需更多历史数据，请切换到日线周期。

### Q5：如何修改 AI 模型？
**A**：菜单 → 设置 → AI 模型设置，在「模型」输入框修改。当前为 `openclaw_wb/deepseek-v4-flash`。
- `openclaw_wb/模型名` —— 走 WorkBuddy 路由（推荐，免 Token）
- `openclaw` —— 走 QClaw 本地网关
- `openclaw_cs` —— 走 Cursor 订阅
- 也可填写自定义模型 + Base URL + API Key

### Q6：夜盘数据不更新
**A**：夜盘交易时段（21:00 后）数据会自动刷新。若品种当夜无夜盘交易，则无新数据。

### Q7：如何查看程序日志
**A**：日志文件位于 `C:\PA_Agent\logs\`：
- `pa_agent.log` —— 运行日志
- `crash.log` —— 崩溃日志

每个期货品种都有**多个具体月份合约**同时在交易。例如氧化铝（AO）当前有约 10-19 个在交易合约（AO2607、AO2608…AO2707）。

### 如何查询某品种当前可交易的具体合约

程序内置了合约查询功能（`list_contracts` 方法）。在 Python 中调用：

```python
from pa_agent.data.eastmoney_futures_source import EastMoneyFuturesSource
src = EastMoneyFuturesSource()
src.connect()
# 查询氧化铝当前可交易的所有合约
contracts = src.list_contracts("AO0")  # 或 list_contracts("AO")
print(contracts)  # 如 ['AO2601','AO2602',...,'AO2707']
src.disconnect()
```

> 该方法会逐月探测（耗时约 10-30 秒），返回所有能获取数据的合约（含已到期可查历史的）。

### 如何在 GUI 中使用具体月份合约

直接在「品种」输入框中输入具体月份代码即可，例如：

- `AO2509` —— 氧化铝 2025年9月合约
- `RB2610` —— 螺纹钢 2026年10月合约
- `AU2512` —— 黄金 2025年12月合约

程序会自动识别 4 位数字（YYMM 格式）并获取该合约的 K 线数据。
