<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbitrary caller permit -> SQLite lease | Any caller can assemble lookalike fields, but only a persisted current proof can be leased. |
| leased SQLite outbound -> gateway dispatch | The coordinator may make its sole gateway call only after atomic lease returns the reconstructed canonical value. |
| post-dispatch transport failure -> reconciliation | A durable lease cannot become another dispatch authorization after an ambiguous outcome. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-36 | Elevation/Tampering | dispatch proof and coordinator | critical | mitigate | Migration-backed exact identity/proof verification and rowcount-one lease occur before the sole gateway call; all forge/replay cases have zero-call regressions. |
| T-02-37 | Repudiation | dispatch lease audit | high | mitigate | Persist one-way proof/lease state with ticket and lifecycle audit facts. |
| T-02-38 | Denial/Elevation | post-lease gateway exception | high | mitigate | Durable lease records ambiguity/reconciliation without reopening or minting a second proof. |
| T-02-SC | Tampering | package installs | low | accept | Existing SQLite/Pytest stack only. |
</threat_model>
