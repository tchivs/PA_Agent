<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| paper operation -> SQLite transaction | A valid typed operation may change paper truth only through one immediate transaction. |
| paper truth -> central ledger | No write occurs in this plan; a later evidence-only projector may read canonical paper facts. |
| restart -> paper store | Reopened state is loaded from paper-owned durable records, never inferred from local memory. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-04 | Tampering | observation cursor/event append | critical | mitigate | Cursor version/digest validation and atomic event/fill/snapshot write prevent duplicate or reordered observation mutation. |
| T-03-05 | Repudiation | cancel/fill race | high | mitigate | One persisted sequence and append-only request/evidence facts decide the outcome. |
| T-03-06 | Tampering | database separation | critical | mitigate | A separate path/schema never reads or treats central projections as paper authority. |
| T-03-07 | Denial of service | SQLite writes | medium | mitigate | Short immediate transactions, bound parameters, indexes/cursors, and no polling loop. |
| T-03-SC | Tampering | package installs | low | accept | No package-manager command is planned. |
</threat_model>
