<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| credentials -> settings/logging/audit | Secret material must never cross into generic persistence or diagnostics. |
| credential reference -> provider | Only opaque reference metadata reaches the lookup port. |
| provider capability result -> execution service | A provider cannot grant or report withdrawal permission before a credential could reach execution-facing code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05 | Information Disclosure | settings, logger, audit and notification payloads | high | mitigate | Reference-only settings plus centralized recursive redaction and synthetic end-to-end leakage tests. |
| T-02-06 | Elevation | credential provider selection | medium | mitigate | Narrow port exposes lookup by opaque reference only; no provider is selected or injected into advisory code. |
| T-02-14 | Elevation | credential capability contract | high | mitigate | Trading-only capability values make withdrawal non-grantable; rejecting synthetic withdrawal declarations before execution consumers proves fail-closed SAFE-05 enforcement. |
| T-02-SC | Tampering | package installs | low | accept | No package installation occurs; existing project dependencies provide the required contracts. |
</threat_model>
