<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller -> public ledger recovery API | Caller input is untrusted and must not convey zero-scope gateway evidence, proof, or transition binding. |
| controlled collector -> SQLite recovery transaction | Gateway account, count, open-order list, connection, and server-time observations are trusted only after collection, canonical parsing, freshness, and transaction-time durable checks. |
| RECOVERING -> READY | A new internal challenge-bound proof must pass persistent one-time/expiry checks before the conditional state mutation. |
| recovery -> authorization/dispatch | Recovery state changes must never mint ticket, command, claim, permit, outbound, or gateway-submit authority. |
