<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| credential material -> observable credential failure | In-memory resolved values must become only redacted data or controlled reason codes. |
| proposal/evidence failure -> SQLite audit | Only allowlisted, redacted canonical summaries may persist. |
| application value/error -> root log, record, or notification payload | Every actual output producer must sanitize before serialization or logging. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-32 | Information Disclosure | credential error boundary | high | mitigate | Register values before observable failure and emit only SecretRedactor output or stable reason codes; real interface tests scan all sinks. |
| T-02-33 | Information Disclosure | SQLite proposal/audit persistence | high | mitigate | Allowlist and redact inside the existing append-only transaction; refuse raw exception and credential objects. |
| T-02-34 | Information Disclosure | root logs, records, Feishu, PushPlus | high | mitigate | Route all actual transaction-related output producers through the same redactor and scan captured logs, payloads, JSON, and CSV. |
| T-02-SC | Tampering | package installs | low | accept | No package-manager action is planned. |
</threat_model>
