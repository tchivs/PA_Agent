<interfaces>
`AnalysisRecord.execution_snapshot` is `ExecutionSafeAnalysisSnapshotV1`, serialized only by `PendingWriter.save_full`. `AnalysisRecordSnapshotReader` is the concrete strict reader: it returns `EligibleAnalysisRecord` with an immutable `SourceAnalysisSnapshot` only for a fully conforming current-schema record, otherwise frozen `IneligibleAnalysisRecord(reason_code, safe_message)`. Compatibility is fail-closed: legacy, incomplete, malformed, old-version, stale, repaired, digest-mismatched, Decimal-invalid, or unsupported records are not migrated, inferred, defaulted, or backfilled. Public façade commands return safe immutable ticket/latch/projection results and never return `OutboundDispatchPermit`, `OutboundSubmission`, gateway, ledger, or raw exceptions.
</interfaces>

| Boundary | Description |
|---|---|
| persisted analysis → approval candidate | advisory storage 的松散历史不能自动获得 execution authority。 |

| T-04-02 | Elevation of Privilege | ticket approval | high | mitigate | strict reader + façade only consumes ticket; existing SQLite permit/lease/coordinator chain and replay/concurrency integration tests are blocking. |
| T-04-03 | Information Disclosure | record/service errors | high | mitigate | controlled eligibility/command DTO only，worker使用 redactor；不返回 raw record/payload/exception。 |
| T-04-04 | Tampering | kill-switch recovery | high | mitigate | façade delegates persisted latch/begin/complete and returns preconditions/blockers; tests prohibit local READY/false cancellation completion。 |
