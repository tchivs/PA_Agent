# PA Agent — 分析速度优化操作指南

> 从操作层面提升程序分析速度的实用方法，含每项的**效果、优缺点和适用场景**。

---

## 速度影响因子全景图

```
分析总耗时 = 阶段一网络请求 + 阶段一AI生成 + 阶段二网络请求 + 阶段二AI生成
                       ↑                    ↑                       ↑                    ↑
                  ┌────┴────┐         ┌─────┴─────┐          ┌────┴────┐         ┌─────┴─────┐
                  │prompt大小│         │思考量×模型 │          │prompt大小│         │思考量×模型 │
                  │K线数×指标│         │速度×effort │          │策略文件数│         │速度×effort │
                  └─────────┘         └───────────┘          └─────────┘         └───────────┘
```

**可操作项**按影响大小排序：

| 排名 | 操作项 | 预计提速 | 速度影响来源 |
|:----:|--------|:--------:|-------------|
| 1 | 关闭 Thinking | ⭐⭐⭐⭐⭐ | 去掉思考 token 生成时间 |
| 2 | 换更快模型 | ⭐⭐⭐⭐⭐ | 模型本身生成速度差异巨大 |
| 3 | 降低 Reasoning Effort | ⭐⭐⭐⭐ | 减少思考深度和 token 数 |
| 4 | 减少 K 线数量 | ⭐⭐⭐ | 减少 prompt 输入 token |
| 5 | 使用增量分析 | ⭐⭐⭐ | 复用上次结论，跳过重复分析 |
| 6 | 精简经验库 | ⭐⭐ | 减少附加的经验案例 token |
| 7 | 调整交易倾向 | ⭐ | 极度激进模式输出更长 |

## 1. 关闭 Thinking（效果最大）

### 操作方式

**设置对话框 → AI 提供商 → Thinking → 关闭**

或直接编辑 `config/settings.json`：
```json
"provider": {
  "thinking": false
}
```

### 速度提升

| 模型 | Thinking 开 | Thinking 关 | 提速幅度 |
|------|:-----------:|:-----------:|:--------:|
| claude-sonnet-4-6 | ~45–90s | ~8–20s | **3–5 倍** |
| deepseek-v4-pro | ~60–120s | ~15–30s | **3–4 倍** |
| claude-opus-4-7 | ~90–180s | ~15–40s | **4–6 倍** |

### 原理

开启 Thinking 时，AI 会先在 `reasoning_content` 区进行长篇推理（数千至数万 token），然后再在 `content` 区输出最终 JSON。关闭后 AI 直接输出 JSON，跳过整个思考过程。

### ✅ 优点

- **提速效果最显著**，通常 3–5 倍
- 大幅减少输出 token 数（省 API 费用）
- 网络传输量减少

### ❌ 缺点

- **分析质量明显下降**：没有深度推理，AI 容易遗漏复杂形态
- 闸门判断可能不够严谨，出现逻辑矛盾
- 决策树的 trace 推理链变短或空洞
- 信心分数和胜率估算可信度降低
- **极端行情下不推荐关闭**（尖峰、过渡期、多空分歧大的场景）

### 适用场景

| 场景 | 推荐 |
|------|:----:|
| 快速扫描多个品种 | ✅ |
| 明显趋势行情只需确认方向 | ✅ |
| 验证某个 setup 是否值得深究 | ✅ |
| 尖峰/过渡期/极端交易区间 | ❌ |
| 大资金实盘决策 | ❌ |
| 首次分析新品种/新周期 | ❌ |

## 2. 更换更快模型

### 操作方式

**设置对话框 → AI 提供商 → 模型**，输入模型名称。

### 常见模型速度对比

| 模型 | 提供商 | 阶段一耗时 | 阶段二耗时 | 分析质量 | 费用 |
|------|--------|:----------:|:----------:|:--------:|:----:|
| **claude-sonnet-4-6** | PackyAPI | ~20–45s | ~25–50s | ⭐⭐⭐⭐⭐ | 中 |
| **deepseek-v4-pro** | DeepSeek | ~30–60s | ~40–70s | ⭐⭐⭐⭐ | 低 |
| **claude-opus-4-7** | PackyAPI/云雾 | ~45–90s | ~60–120s | ⭐⭐⭐⭐⭐ | 高 |
| **deepseek-chat** | DeepSeek | ~10–20s | ~15–25s | ⭐⭐⭐ | 极低 |
| **claude-sonnet-4-6（关思考）** | PackyAPI | ~8–20s | ~10–25s | ⭐⭐⭐⭐ | 中低 |

### ✅ 优点

- 选对模型可同时获得速度和质量的平衡
- DeepSeek 系列性价比最高
- 不同 API 提供商的转发速度差异也很大（PackyAPI 通常快于云雾）

### ❌ 缺点

- 更快的模型通常质量更低
- 切换模型后需确认 API 兼容性（Base URL + API Key 配套）
- 部分模型不支持 Thinking（如 deepseek-chat）
- 模型切换需要重新测试输出格式兼容性

### 推荐组合

| 需求 | 推荐模型 + 配置 |
|------|----------------|
| **日常实盘** | claude-sonnet-4-6 + Thinking 开 + effort=high |
| **快速扫描** | deepseek-chat + Thinking 关 |
| **深度复盘** | claude-opus-4-7 + Thinking 开 + effort=max |
| **性价比** | deepseek-v4-pro + Thinking 开 + effort=medium |

## 3. 降低 Reasoning Effort

### 操作方式

**设置对话框 → AI 提供商 → Reasoning Effort**

可选值：`max` / `high` / `medium` / `low`

### 速度影响

| Effort | 思考 token 量（约） | 阶段总耗时 | 适用 |
|:------:|:-------------------:|:----------:|------|
| max | 8000–20000 | 基准 100% | 深度分析 |
| high | 4000–10000 | ~70% | 日常分析 |
| medium | 2000–6000 | ~50% | 快速确认 |
| low | 500–2000 | ~30% | 极速扫描 |

> ⚠️ 具体数值因模型和行情复杂度差异很大，上表为粗略估算。

### ✅ 优点

- 不需要关闭 Thinking，仍保留推理能力
- 渐进式调速：可以从 high 开始尝试，不够快再降
- 省输出 token（费用降低）

### ❌ 缺点

- effort 过低时思考可能不够深入，遗漏关键结构
- `low` 档位 AI 可能跳过重要闸门节点
- 部分模型（DeepSeek 原生 API）的 effort 实际效果不如 Claude 系列
- PackyAPI 的 Claude 模型使用 `budget_tokens` 而非 `effort`，效果略有差异

### 适用场景

- 日常使用推荐 **high**（比 max 快约 30%，质量差距不大）
- 快速确认推荐 **medium**
- **不建议用 low**（除非只是看看 AI 有没有明显方向）

## 4. 减少 K 线数量

### 操作方式

**主界面控制栏 → K线数**，拖动或直接输入数值（2–5000）。

### 速度影响

K线数量直接影响 **prompt 输入 token 数**：

| K线数 | 输入 token（约） | 阶段一耗时影响 | 分析覆盖范围 |
|:-----:|:----------------:|:--------------:|-------------|
| 30 | ~3,000 | 最快 | 仅近期结构 |
| 50 | ~5,000 | 较快 | 基本结构 |
| 100 | ~10,000 | 基准 | 标准覆盖 |
| 200 | ~20,000 | 慢 ~20% | 扩展结构 |
| 500 | ~50,000 | 慢 ~50% | 大周期复盘 |

### ✅ 优点

- 操作最简单，无需改设置
- 同时降低输入 token 费用
- 减少超时风险

### ❌ 缺点

- **K线太少时 AI 看不到完整结构**，诊断可能误判
- 小周期（1m/5m）50 根可能只覆盖几十分钟，缺乏上下文
- 阶段一的 `bar_by_bar_summary` 最多输出 12 根，K线过少意义不大
- 大周期（4h/1d）100 根已覆盖数周，减少影响较小

### 推荐数量

| 周期 | 推荐 K线数 | 说明 |
|------|:---------:|------|
| 1m–5m | 80–150 | 需要足够K线识别微观结构 |
| 15m | 60–100 | 默认值即可 |
| 1h | 50–100 | 100 根 ≈ 4 天，足够 |
| 4h | 50–80 | 50 根 ≈ 8 天 |
| 1d | 30–50 | 50 根 ≈ 2 个月 |

## 5. 使用增量分析

### 操作方式

- **自动增量**：当新增K线 ≤ 阈值时，点「提交分析」自动走增量
- **强制增量**：点「增量分析」按钮
- 阈值配置：设置 → 通用 → 增量分析最大新增K线（默认 10）

### 速度影响

| 模式 | 说明 | 提速幅度 |
|------|------|:--------:|
| 完整分析 | 从零开始两阶段分析 | 基准 |
| 增量分析 | AI 可参考上次结论 | **20–40%** |
| 无新增K线增量 | 仅复核上次结论 | **40–60%** |

### ✅ 优点

- **不牺牲质量**：AI 仍走完整决策树，只是能参考上次分析
- 节省输入 token（上次结论压缩了推理空间）
- 适合持续盯盘场景

### ❌ 缺点

- 首次必须先做一次完整分析
- K线缺口过大时无法对齐，增量失败
- 增量分析可能"继承"上次的偏见
- 强制增量时如果上次记录有误，错误会传导

### 适用场景

| 场景 | 推荐 |
|------|:----:|
| 持续盯盘，每收盘一根新K线 | ✅ |
| 同品种周期反复分析 | ✅ |
| 首次分析 | ❌（无法增量） |
| 隔了很久再看 | ❌（缺口太大） |

## 6. 精简经验库

### 操作方式

减少 `experience/` 目录下的案例文件数量。

每个子目录（如 `normal_channel/success_cases/`）中的 `.json` 文件都会被加载到 prompt。

### 速度影响

| 经验条目数 | 附加 token（约） | 耗时影响 |
|:---------:|:----------------:|:--------:|
| 0 | 0 | 基准 |
| 1–2 | ~1,000 | +3–5% |
| 3–5 | ~3,000 | +8–12% |
| 6–10 | ~6,000 | +15–20% |

### ✅ 优点

- 减少输入 token
- 可选择只保留最有价值的案例

### ❌ 缺点

- 经验案例帮助 AI 参考历史类似行情，减少可提高一致性但降低参考性
- 需要手动管理文件
- 对整体速度影响不如前几项显著

### 建议

- 保留每个 `cycle_position` 下 **1–2 个最佳案例**即可
- 删除过于相似或信息量低的案例

## 7. 调整交易倾向

### 操作方式

**设置 → 通用 → 交易倾向**

### 速度影响

| 倾向 | 输出长度 | 耗时影响 |
|------|:--------:|:--------:|
| 保守 | 正常 | 基准 |
| 均衡 | 正常 | 基准 |
| 激进 | 略长 | +5–10% |
| 极度激进 | 较长 | +10–15% |

### 原因

极度激进模式下，AI **被强制要求**给出具体做多/做空方案，即使信号不完美也要写出完整 entry/stop/target 和 reasoning，输出 token 更多。

### ✅ 优点

- 极度激进保证一定能得到交易方案
- 不太影响总体速度

### ❌ 缺点

- 极度激进可能给出质量不高的交易建议
- 速度改善微乎其微，不推荐为提速而调整此项

## 综合提速方案

### 方案 A：极速扫描模式（8–15 秒出结果）

```
模型:     deepseek-chat 或 claude-sonnet-4-6
Thinking: 关闭
Effort:   —（关闭 Thinking 后无效）
K线数:    50
增量:     开启（阈值 10）
经验库:   保留 1–2 条
倾向:     均衡
```

**用途**：快速浏览多个品种，判断哪些值得深入分析。

**代价**：诊断精度约 70–80%，可能误判复杂结构。

---

### 方案 B：日常实盘模式（20–40 秒出结果）

```
模型:     claude-sonnet-4-6
Thinking: 开启
Effort:   high
K线数:    80–100
增量:     开启（阈值 10）
经验库:   保留 3–5 条
倾向:     均衡
```

**用途**：日常实盘分析，速度与质量的最佳平衡。

**代价**：比 max effort 略浅的推理，极复杂行情可能遗漏。

---

### 方案 C：深度复盘模式（60–180 秒出结果）

```
模型:     claude-opus-4-7
Thinking: 开启
Effort:   max
K线数:    150–200
增量:     关闭（完整分析）
经验库:   全部保留
倾向:     保守
```

**用途**：复盘关键行情、验证策略、重要决策。

**代价**：耗时较长，token 费用高。

---

### 方案 D：持续盯盘模式（15–25 秒/次更新）

```
模型:     claude-sonnet-4-6
Thinking: 开启
Effort:   medium
K线数:    60–80
增量:     强制开启（阈值 20）
经验库:   保留 2–3 条
倾向:     均衡
```

**用途**：每收盘一根新K线后增量更新，快速跟踪。

**代价**：依赖首次完整分析的准确性。

## 速查决策树

```
你现在最需要什么？
│
├─ 快速看方向 → 方案A（极速扫描）
│   └─ 关 Thinking + 快模型 + 少K线
│
├─ 日常分析 → 方案B（日常实盘）
│   └─ Thinking 开 + effort=high + 增量
│
├─ 重要决策 → 方案C（深度复盘）
│   └─ 强模型 + effort=max + 多K线
│
└─ 盯盘跟踪 → 方案D（持续盯盘）
    └─ effort=medium + 强制增量 + 等待收盘
```

## 常见误区

| 误区 | 正解 |
|------|------|
| "关 Thinking 等于白用 AI" | 关闭后仍有基本分析能力，简单行情下质量可接受 |
| "模型越快越好" | deepseek-chat 极快但分析质量明显低于 claude 系列 |
| "K线越多越好" | 超过 200 根后边际收益极低，徒增耗时和费用 |
| "effort=low 够用" | low 档闸门判断经常不完整，容易漏掉关键节点 |
| "增量分析质量低" | 增量分析仍走完整决策树，质量与完整分析几乎一致 |
| "换模型要改代码" | 只需在设置中改模型名和 Base URL，无需改代码 |

## 配置变更位置速查

| 参数 | 设置对话框路径 | JSON 路径 |
|------|---------------|-----------|
| Thinking | AI 提供商 → Thinking | `provider.thinking` |
| Reasoning Effort | AI 提供商 → Reasoning Effort | `provider.reasoning_effort` |
| 模型 | AI 提供商 → 模型 | `provider.model` |
| Base URL | AI 提供商 → Base URL | `provider.base_url` |
| 用于分析的 K 线数量 | 设置 → 通用设置 | `general.analysis_bar_count`（默认 100） |
| 增量阈值 | 通用 → 增量分析最大新增K线 | `general.incremental_max_new_bars` |
| 交易倾向 | 通用 → 交易倾向 | `general.decision_stance` |
| 上下文窗口 | AI 提供商 → Context Window | `provider.context_window` |
