<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller to canonical domain | Untrusted runtime objects must not impersonate enums, contexts, evidence, or numeric rules. |
| caller to ledger | A caller can request logical admission but cannot choose the durable remote identity or raw observation payload. |
| ledger to future gateway | The only future submit authority is an irreversible ledger-created outbound authorization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-GC-01 | Tampering | canonical constructors | high | mitigate | Runtime type checks, discriminated ProductContext validation, strict Decimal ingress, non-negative minima, and typed observation records reject malformed data. |
| T-01-GC-02 | Spoofing | durable client identity | critical | mitigate | The port makes the ledger the sole allocator; caller candidates are never durable remote identities. |
| T-01-GC-03 | Elevation | outbound admission | critical | mitigate | A protected outbound authorization replaces the stale read-before-submit contract; adapters accept only the protected result. |
| T-01-GC-04 | Information disclosure | account observations | high | mitigate | Persistence accepts only explicit canonical balances/positions and ProductType scope, excluding arbitrary/raw/secret payload shapes. |
| T-01-SC | Tampering | package installation | low | accept | No package installation is planned. |
</threat_model>
