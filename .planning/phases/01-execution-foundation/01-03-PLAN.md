<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| domain repository request to SQLite | Canonical command and observation values cross into durable local storage. |
| local process to filesystem | The ledger path, permissions, and SQLite configuration become operational controls. |
| process restart to ledger reopen | Non-terminal work must retain its identities and admission state. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04 | Tampering | command/admission rows | critical | mitigate | Unique logical/client/claim constraints and one transaction ensure only one unresolved command can receive an admissible claim. |
| T-01-05 | Repudiation | order event history | high | mitigate | Append-only sequenced events, stable IDs, and projections generated only by ledger methods preserve evidence. |
| T-01-06 | Tampering | SQLite configuration and payload fields | high | mitigate | Verify required PRAGMAs, use parameter binding/canonical serialization, and fail closed instead of weaker operational settings. |
| T-01-07 | Denial of service | local SQLite writer | medium | mitigate | Single-process connection ownership, short transactions, a 5000 ms timeout, and typed lock failures avoid indefinite blocking. |
| T-01-08 | Information disclosure | execution database | medium | mitigate | Restrictive portable permissions, sanitized canonical fields, and no secrets or raw private payloads. |
| T-01-SC | Tampering | package installation | low | accept | No package installation is planned; sqlite3 is in the standard library. |
</threat_model>
