<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| UI command request → durable approval/kill-switch services | 操作员动作只能请求既有服务，不能获取执行权威。 |
| reopened durable storage → workspace projection | 重开事实必须真实且只读，不得触发副作用。 |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-04-01 | Tampering | asynchronous target state | high | mitigate | 后续 worker tests强制 generation/target digest/UI-alive guards；配置 target 修改立即撤销旧 readiness。 |
| T-04-02 | Elevation of Privilege | ticket command façade | high | mitigate | 集成测试验证 consume → permit → lease → coordinator 的唯一链路、并发/过期/revoked 无第二 authority。 |
| T-04-03 | Information Disclosure | visible errors | high | mitigate | 配置测试只接受 controlled Chinese validation/error DTO；实现将由 SecretRedactor 清洗 worker error。 |
| T-04-04 | Tampering | recovery/lifecycle UI | high | mitigate | reopen/kill-switch tests要求 persisted latch、cancellation evidence和服务 preconditions，禁止 local READY。 |
| T-04-05 | Denial of Service | Qt request dispatch | high | mitigate | 后续 worker suite覆盖 delayed operation；本计划 GUI contract 不允许同步 service invocation。 |
</threat_model>
