<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator approval -> ledger admission | A UI/operator action must become one durable authorization only after current verification. |
| ticket persistence -> outbound submission | Ticket status, hashes, expiry, and kill state must be checked in the same transaction that starts outbound authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Repudiation/Elevation | ticket consumption and `outbound_started` | high | mitigate | Conditional ticket consumption, hash/expiry/kill checks, command binding, and outbound start share one immediate SQLite transaction under D-11. |
| T-02-10 | Tampering | stale/replayed approval evidence | high | mitigate | Fresh evidence/risk re-evaluation, fixed TTL, immutable binding hashes, and D-10 invalidation tests block stale approvals. |
| T-02-11 | Repudiation | operator terminal actions | medium | mitigate | Distinct append-only terminal events preserve reason, timestamp, actor label, and binding snapshot under D-12. |
| T-02-13 | Tampering | fee estimate and review summary | high | mitigate | The target/product/symbol/quote/rate-version-bound Decimal fee estimate is hashed into evidence, assessment, and ticket; missing, stale, or mismatched rates reject before ticket issue or consumption. |
| T-02-SC | Tampering | package installs | low | accept | No dependency installation is part of this plan. |
</threat_model>
