# Phase 4 — 本地交易工作区 UI 设计契约

> 本契约是 Phase 4 的视觉、交互与线程边界来源。它复用现有 PyQt6 深色交易终端主题；工作区只投影已持久化的交易事实并请求既有 application service，绝不成为网关、风险、审批、生命周期或对账权威。

---

## 设计系统

| 属性 | 契约值 |
|---|---|
| 工具 | 无；这是 PyQt6 Widgets 桌面应用，不适用 shadcn 初始化门槛 |
| 组件库 | PyQt6 Widgets；复用 `QTabWidget`、`QSplitter`、`QGroupBox`、`QTableWidget/QTableView`、`QDialog` 与 `QMessageBox` |
| 图标库 | 无新增图标库；仅使用 Qt 标准图标和带中文文字的按钮，风险动作不得只用图标表达 |
| 字体 | `"Segoe UI", "Microsoft YaHei UI", sans-serif`；数值、时间、订单/票据标识使用 `"JetBrains Mono", "Cascadia Mono", "Consolas", monospace` |
| 现有主题 | 复用 `pa_agent/gui/theme/tokens.py` 与 `dark.qss`；Fusion 风格、6px 圆角、8px 基础间距、深色数据表 |
| 来源 | `04-CONTEXT.md`、`04-RESEARCH.md`、`pa_agent/gui/theme/tokens.py`、`pa_agent/gui/theme/dark.qss`、`main_window.py` |

**组件规则：**新增交易面板沿用 purpose-named sibling 模块模式；`MainWindow` 只增加“交易工作区”入口、挂载和关闭转发。复用已存在的暗色表格 palette、交替行、整行单选、只读单元格和 `QHeaderView` 列宽策略。不要引入第二套 CSS、第二种深浅主题、渐变、玻璃效果或装饰性动画。

## 间距刻度

声明值均为 4 的倍数：

| Token | 值 | 用途 |
|---|---:|---|
| xs | 4px | 图标与文字间距、状态标签内微间距 |
| sm | 8px | 表单字段、状态条、表格标题与元数据间距；现有基础间距 |
| md | 16px | 面板默认内边距、相邻控件组间距 |
| lg | 24px | 页面主区域内边距、配置分区之间的间距 |
| xl | 32px | 大型分栏与一级内容区间距 |
| 2xl | 48px | 空状态图文与主要下一步操作之间的留白 |
| 3xl | 64px | 不使用于密集工作区；仅保留为应用级页面间距 token |

**强制对齐：**本阶段不存在间距例外。所有间距、内边距、间隙和拖拽柄必须只使用 `4px`、`8px`、`16px`、`24px`、`32px`、`48px` 或 `64px`。`pa_agent/gui/theme/tokens.py` 与 `pa_agent/gui/theme/dark.qss` 的规范主题规则必须对齐此刻度；现有规则不得被当作例外。
- 所有鼠标可点击按钮、下拉框、复选框和表格行的可操作高度至少为 **32px**；“保存并验证”“确认批准并提交”“触发全局熔断”“开始恢复”同样至少 **32px**。
- 顶部连续状态条与表格行保持紧凑：状态标签内边距 `4px 8px`，表格单元格 `4px 8px`。
- `QSplitter` 使用 `4px` 拖拽柄；禁止用无意义的卡片嵌套制造分区。

## 字体排版

只使用以下四个字号和两种字重：

| 角色 | 字号 | 字重 | 行高 | 用途 |
|---|---:|---:|---:|---|
| 元数据 | 11px | 400 | 1.45 | 时间戳、来源、数据新鲜度、辅助说明、状态栏 |
| 正文/表格 | 13px | 400 | 1.5 | 表单值、按钮、表格正文、说明性错误文本 |
| 标签/小标题 | 16px | 600 | 1.25 | `QGroupBox` 标题、产品区标题、审批单字段分组 |
| 页面标题 | 20px | 600 | 1.2 | “交易工作区”、确认对话框标题、不可恢复风险提示 |

- 数值列（余额、仓位、价格、数量、费用、杠杆、时间）右对齐并使用等宽字体；金额保留 domain projection 提供的精度，UI 不自行四舍五入为业务事实。
- 仅使用 400 与 600；不以全大写、紧缩字距或颜色单独承载关键信息。
- 表头固定 13px/600，正文 13px/400。辅助说明允许换行，正文说明最大 75 个中文字符左右后换行；票据 ID、摘要和路径可水平滚动或用省略号呈现，并提供完整 tooltip/复制操作。

## 色彩

| 角色 | 值 | 使用范围 |
|---|---|---|
| 主导色（60%） | `#0a0e14` | 应用背景、表格基底、输入区域 |
| 次级色（30%） | `#161b22`；抬升层 `#1c2128` | 标签页内容、工具栏、分组容器、交替表格行、对话框 |
| 强调色（10%） | `#38bdf8` | 当前标签页下划线、键盘焦点、已选表格行、当前产品/当前目标的非风险性选中状态 |
| 成功 | `#22c55e`，主操作背景可沿用 `#15803d` | 已应用且就绪、对账新鲜、成功保存；“保存并验证”主按钮 |
| 警告 | `#f59e0b` | 草稿未保存、数据将过期/已过期、非阻断性提示、Testnet 不可用说明 |
| 危险 | `#ef4444` | 阻断错误、LATCHED 状态、触发熔断、拒绝/终止性失败；仅破坏性动作 |
| 信息/辅助 | `#2dd4bf`、`#8b949e` | 只读来源与辅助状态；不得取代成功、警告或危险颜色 |

**强调色保留给：**当前标签页、聚焦输入、选中表格行、当前 product/target 选择。它**不得**用于“可批准”“已提交”“已恢复”或熔断危险动作。

**语义状态：**复用现有 `pillGreen`、`pillAmber`、`pillRed`、`pillBlue`、`pillCyan` 外观；每个彩色 pill 必须同时有可见文字，例如“已就绪”“草稿未保存”“数据已过期”“已锁存”。文字对其表面必须达到正常字号至少 4.5:1 对比度；焦点以 `#38bdf8` 1px 边框显示，不得只以颜色区分。

## 信息架构与页面布局

### 顶层入口与默认页

1. `MainWindow` 菜单栏新增单一文字入口 **“交易工作区”**；不在分析、告警、通知、`DecisionPanel` 或任何通知面板添加提交控件。
2. 打开后使用 `QTabWidget` 承载三个一级页：**“账户状态”**（默认）、**“交易配置”**、**“审批单”**。沿用现有选中 tab 的青蓝下划线、未选中灰字和 6px 圆角。
3. 三页之上固定一条紧凑状态带，顺序固定为：**连接**、**对账**、**配置就绪**、**熔断**。账户状态仍是首屏主体，状态带不得挤占主要余额/仓位区域。
4. 状态带内容来自 immutable workspace projection，显示文本、状态 pill、最后更新时间和 tooltip；不从图表、CSV、alert 或 widget 缓存推断交易事实。

### 桌面分栏与窗口适配

| 可用窗口宽度 | 布局规则 |
|---|---|
| ≥1280px | “账户状态”使用垂直 `QSplitter`：上方只读跨产品摘要，下方产品标签页；产品页内左侧余额/仓位、右侧未结订单/近期成交，默认 3:2 stretch。 |
| 1024–1279px | 保留产品标签页；产品页改为垂直上下区，余额/仓位在上、订单/成交在下；表格保持水平滚动，不压缩数字精度。 |
| <1024px 或窗口高度 <700px | 不创建移动版；将左右 `QSplitter` 折叠为单列可滚动区域，表格保持最小列宽并提供水平滚动。固定状态带与“保存并验证”操作保持可见，次要来源详情放入展开区。 |

- 初始 trading workspace 最小可用尺寸为 **1024×700**；窗口更小时功能不隐藏，只通过分栏重排、滚动与列优先级保证可访问。
- 表格列优先级：产品、资产/合约、可用/仓位、方向、数量、状态始终显示；来源、长标识和详细原因在窄窗口移入行详情/tooltip，不截断安全状态。
- 拖拽分栏位置在本次会话保留；窗口缩小时禁止压缩按钮文本或让中文文本溢出。所有长文本 label 使用 word-wrap，表格 ID 使用省略号 + 完整 tooltip。

### 账户状态：按产品分组且只读

- 跨产品摘要位于账户页顶部，标题固定为 **“跨产品概览（仅供查看）”**。它仅显示各产品账户数、资产/仓位项目数、未结订单数、最近成功对账时间范围；显式显示文案 **“此概览不计算风险，也不决定是否可审批。”**
- 下方 `QTabWidget` 固定顺序：**“现货”**、**“逐仓杠杆”**、**“USDT 永续”**。不可用产品页仍显示，内容为 capability projection 的不可用原因，不能伪装为零余额。
- 每个产品页按此顺序包含独立只读区：**余额**、**持仓**、**未结订单**、**近期成交**、**网关错误与对账**。每个区标题右侧独立显示 `来源 · 上次成功对账 · 新鲜度`；不得由摘要复用或覆盖。
- 所有交易表使用不可编辑 `QTableView/QTableWidget`、交替行、整行单选。最少列如下：

| 区域 | 必须列 |
|---|---|
| 余额 | 资产、总额、可用、冻结/占用、来源、上次成功对账、新鲜度 |
| 持仓 | 合约/资产、方向、数量、均价、保证金/借贷上下文、杠杆（如适用）、来源、新鲜度 |
| 未结订单 | 客户订单 ID、标的、方向、数量、已成交/剩余、状态、提交/观测时间、对账状态 |
| 近期成交 | 成交 ID、订单 ID、标的、方向、数量、价格、费用、成交时间、来源 |
| 网关错误与对账 | 时间、操作、受控错误码、对账状态、下一步；禁止显示原始异常、请求头、payload 或 secret |

## 交互契约

### 交易配置：渐进显示、草稿与已应用快照

1. 配置页使用可滚动 `QGroupBox` 渐进区，不显示与当前选择不相干的字段。顶部始终并列显示两个不可混淆的摘要：**“当前已应用配置”**和**“正在编辑的草稿”**。
2. 选择顺序固定：**模式 → 场所/环境 → 产品 → 账户**；仅在此四项选择后显示适用的 **标的映射**、**Paper 余额**、**逐仓/杠杆控制**、**风险限额**、**非秘密连接设置**分区。
3. 默认模式为 **“Paper（默认）”**。**“Testnet（需单独配置，当前阶段不可用）”**可见但不可进入就绪状态；**“Live（已禁用）”**可见、不可编辑、不可保存、不可出现 endpoint、credential 选择或提交路径。
4. 编辑场所、环境、账户或产品时：保留所有未保存值为草稿；立即重验受影响字段；失效集中 readiness；审批入口立即不可用，直到新草稿成功保存并验证。不得丢弃草稿或默默回退成已应用值。
5. 每个字段在编辑后显示本地格式/必填提示；最终可用性只能由后台 façade 返回的 section-keyed validation/readiness projection 确定。UI 不能自行宣告风险、能力或审批合格。
6. 底部固定操作栏只有一个主操作：**“保存并验证”**。只有 Pydantic-validated、non-secret settings 和服务验证通过后，草稿才替换“当前已应用配置”。保存失败时已应用快照保持不变。
7. credential 仅能以不透明引用的“已配置/未配置”状态展示；不呈现、复制、存储或回显秘密、签名、header、原始 endpoint secret 或原始 gateway exception。
8. “风险限额”默认显示服务返回的基线与本次草稿的受控值；只有 application 层证明“只收紧、不放宽”时才可编辑。没有该 typed API 时值必须只读并标为 **“由既有安全策略管理”**。

### 集中 readiness

- 配置页右侧（窄窗口时位于固定操作栏之上）为固定 **“审批就绪状态”**摘要；这是当前已应用 target 能否进入审批流的**唯一全局信号**。
- 摘要逐项列出：状态（就绪/不可用/阻断）、问题等级（阻断/警告）、所属分区、受控原因、下一步。每条原因可将焦点带到对应分区。
- 字段局部绿/黄/红提示只能说明该字段；不得取代或冲突于中央 summary。草稿存在、任何阻断、对账过期、熔断非 READY 或目标不可用时，summary 必须显示“不可进入审批流程”。
- “当前已应用配置”与“草稿未保存”同屏可见；草稿状态使用黄色 pill **“草稿未保存，尚未应用”**，已应用且验证通过用绿色 pill **“已应用并通过验证”**。

### 新鲜度、错误与刷新

- 每个产品数据区独立显示最后一次**成功**对账的本地化时间、数据来源和 freshness：**“新鲜”**、**“已过期”**、**“刷新失败”**、**“从未成功对账”**。
- 已过期数据保留可读，并在其标题和相关审批入口显示黄色明确提示；不得清空成零值、不得显示为“已就绪”、不得使 UI 将其用于批准资格。
- **“刷新账户数据”**仅请求后台 façade 的读取/对账动作；加载期间保留最后成功投影并在标题显示 **“正在后台刷新；以下为上次成功数据”**，而不是用空白替换数据。
- 受控错误显示错误码/安全描述/可行动作，重试必须显式由操作员触发。worker 错误经 `SecretRedactor` 处理后才能到 UI；永不渲染 raw traceback 或私有 payload。

### 审批单：严格票据审阅而非提交捷径

1. “审批单”页默认显示 eligible/ineligible 的**持久化分析记录**列表；创建按钮固定为 **“从合格记录创建审批单”**。只有严格 `CompletedAnalysisSnapshotReader` 接受的记录可启用它；缺字段、已过期、已修复、digest 不匹配、缺 Decimal 数量/风险依据或不支持产品时显示不可合格原因，禁止默认补值。
2. 票据审阅使用专用 `QDialog`，标题 **“审批单复核”**。必须完整、只读显示：场所、环境、账户、产品、标的、方向、数量、杠杆或借贷上下文、预计价格与滑点、预计费用、数据年龄、source analysis provenance、风险闸门结果、票据 ID、状态和到期时间；关闭/返回操作固定为 **“返回审批单”**，不得使用无上下文的关闭文案。
3. 审阅页底部明确显示回溯信息 **“来源分析：{安全的来源标识}；此审批单只对应该不可变记录。”**。修改任何绑定输入、票据过期、freshness 变化、被拒绝/撤销或服务返回新风险结果时，关闭旧票据并重新读取持久化状态；UI 不假设成功或终态。
4. 拒绝操作为次级危险操作 **“拒绝审批单”**，先显示确认文案后调用现有 `ApprovalService.reject_ticket`；拒绝后返回只读终态。
5. 批准按钮文案固定为 **“确认批准并提交”**，只在已应用配置就绪、目标 capabilities 允许、数据新鲜、熔断为 READY 且票据仍有效时可用。点击后出现最终确认；该对话框的非主操作固定为 **“不提交审批单”**，不得使用通用“取消”。点击 **“不提交审批单”** 或按 Esc 都只关闭该最终确认对话框，票据及所有服务状态保持不变，不调用任何命令。仅在确认后才在后台按既有 `consume_ticket → permit/lease → SubmissionCoordinator` 路径执行；dialog 不构造 order、permit、OutboundSubmission，不调用 gateway。
6. 告警、通知、分析 stream、`DecisionPanel` 和跨产品摘要不出现“提交”“批准”或等价的交易命令控件。

### 熔断：持续可见、显式确认、服务为准

- 顶部状态带始终显示一个文字 pill：**“熔断：READY”**、**“熔断：LATCHED”**或**“熔断：RECOVERING”**；LATCHED 使用危险色，RECOVERING 使用警告色。它来自持久化 projection，不由窗口本地 bool 决定。
- 按钮 **“触发全局熔断”**只在 READY 可用，使用既有 `dangerButton` 外观。确认对话框标题 **“确认触发全局熔断”**，必须展示当前持久化状态、会阻止新订单、会请求取消可取消未结订单、取消请求不等于已取消；操作员勾选 **“我已知晓新订单将被阻止，未结订单仍需对账确认。”** 后才可点击 **“确认触发熔断”**。关闭确认对话框的次操作固定为 **“不触发熔断”**，不得使用无上下文的关闭文案。
- LATCHED/RECOVERING 状态只显示 **“查看恢复条件”**或 **“开始恢复”**。恢复对话框显示服务返回的 persisted preconditions、待对账 scope、取消/订单证据和所有 blocking reasons；每条 blocker 有中文解释。服务未允许时恢复确认按钮禁用，UI 绝不把状态直接设为 READY。关闭/返回恢复对话框的操作固定为 **“继续查看恢复条件”**，不得使用无上下文的关闭文案。
- 对持久化 command 的点击只启动后台请求；即便用户切换目标或关闭窗口，已进入 durable service 的动作不回滚。下次成功 projection 负责收敛状态。

## 文案契约

所有以下面向操作员的文字必须使用简体中文；`{…}` 仅代表安全、已格式化的 projection 值。

| 元素 | 固定文案 |
|---|---|
| 顶层入口 | `交易工作区` |
| 默认页 | `账户状态` |
| 配置主操作 | `保存并验证` |
| 草稿状态 | `草稿未保存，尚未应用` |
| 已应用状态 | `已应用并通过验证` |
| 集中 readiness 标题 | `审批就绪状态` |
| readiness 不可用 | `当前配置不可进入审批流程` |
| 跨产品摘要标题 | `跨产品概览（仅供查看）` |
| 跨产品摘要说明 | `此概览不计算风险，也不决定是否可审批。` |
| 加载账户 | `正在后台刷新；以下为上次成功数据` |
| 空账户标题 | `暂无账户数据` |

## 可访问性与键盘契约

- 所有控件使用标准 Qt focus 顺序；Tab/Shift+Tab 遍历，Space 切换复选框，Enter 只触发当前聚焦且已启用的安全动作。危险确认不绑定默认 Enter；焦点初始停在当前对话框的上下文关闭操作：审批单为“返回审批单”、审批最终确认为“不提交审批单”、熔断为“不触发熔断”、恢复为“继续查看恢复条件”。
- 每一个文字输入、下拉框、表格和按钮都有简体中文 `accessibleName` 与 `accessibleDescription`；状态 pill、图标和颜色均有同等文字状态。
- 字段错误通过可见文本、tooltip 与关联的 section error label 同时表达；必须将焦点带到首个阻断字段，不能仅依赖红色边框。
- 对账、刷新账户数据、提交、取消订单和恢复的进行状态通过按钮禁用、状态文本和 `QProgressBar`/busy 状态表明；不使用闪烁动画。状态变化如使用淡入，仅限 150–200ms，且关闭系统动画时立即呈现。
- 表格支持键盘行选择、列头可访问名称、水平/垂直滚动；数值不会仅以颜色表达涨跌、风险或订单状态。
- 对话框为真正 modal 的风险确认步骤。审批最终确认中，按 Esc 或点击“不提交审批单”只关闭该最终确认对话框，不调用命令，且不改变 ticket 或任何服务状态；其他对话框按 Esc 或其上下文关闭操作同样不触发命令，关闭 dialog 不改变服务状态。

## 后台工作与陈旧回调安全契约（NFR-01）

1. **禁止 UI 线程交易 I/O。**连接、配置验证、保存、账户/订单/成交读取、对账、approval ticket 创建/拒绝/消费、提交、取消、熔断恢复、SQLite 查询和任何 gateway 调用都必须由 `QThread` worker 调用 worker-owned workspace façade。Qt 主线程只收 queued signal 并渲染 immutable result/safe error DTO。
2. 每次刷新、验证、ticket 命令、熔断命令与 target/product 切换都分配不可变 **generation** 和 **active target digest**。结果 slot 在接触任意 widget、重新启用按钮或更新状态前，必须同时验证：generation 相等、target digest 相等、面板/窗口仍存活且未关闭。
3. 切换 target/product 或关闭工作区的顺序固定为：先失效 generation 和 active target；断开 UI slots；对可中断读取设置 `CancelToken`；不等待阻塞 I/O；追踪尚在运行 worker 为 zombie，稍后 reaping。不得在 UI thread 用有时限的 `wait()` 冻结窗口。
4. 已经进入 durable application service 的提交、取消或恢复不得因 UI cancel 而被假装撤销。过期 callback 只丢弃 UI 结果；后续 workspace projection 显示持久化事实。
5. 任一 worker 失败只发安全错误 DTO；缺少、延迟、失败或过期结果都不得把新 workspace 重新标为 READY、重新启用批准，或覆盖已切换 target 的内容。
6. worker 不持有 `QWidget`，panel 不持有 `TradingGateway`、`PaperStore`、SQLite connection 或 ledger authority。通过 `AppContext` 组合单一 façade；alert/notification 不获得 façade 的消费/提交能力。
