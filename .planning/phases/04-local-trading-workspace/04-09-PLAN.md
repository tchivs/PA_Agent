<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| typed ticket/latch DTO → modal confirmation | dialog 必须显示事实，但不能把显示变成新的 execution/recovery authority。 |
| operator confirmation → worker command façade | 确认行动只能进入既有 durable services。 |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-04-01 | Tampering | delayed dialog callback | high | mitigate | worker generation/digest/UI-alive guards; dialog re-reads persisted projection after command and test switch/close late outcomes。 |
| T-04-02 | Elevation of Privilege | approve control | high | mitigate | no gateway/order/permit constructors in dialog; command façade enforces consume → permit/lease → coordinator and e2e/integration tests are blocking。 |
| T-04-03 | Information Disclosure | ticket/recovery errors | high | mitigate | only safe DTO fields and redacted messages displayed; no raw payload/headers/traceback/credential content。 |
| T-04-04 | Tampering | local recovery/terminal display | high | mitigate | dialog renders persisted latch/cancellation/preconditions; no local READY or terminal cancellation inference; reopen tests protect state。 |
| T-04-05 | Denial of Service | modal command execution | high | mitigate | modal actions enqueue worker only, busy feedback avoids duplicate clicks, pytest-qt checks event-loop responsiveness。 |
</threat_model>
