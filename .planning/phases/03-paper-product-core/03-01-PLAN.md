<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| scenario input -> canonical observation | Untrusted scenario values must be finite Decimal, bound to one scope, and version/digest valid. |
| observation -> matcher | A matcher may derive fill candidates but cannot mutate accounts, ledger, or dispatch authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-01 | Tampering | market observation/depth | high | mitigate | Frozen typed Decimal observations, canonical digest, and monotonic version validation prevent forged or malformed depth from producing a candidate. |
| T-03-02 | Repudiation | fill economics | high | mitigate | Each candidate retains observation identity/version, raw price, fee/slippage Decimal inputs, and rule versions. |
| T-03-03 | Denial of service | matcher ordering | medium | mitigate | Reject invalid or noncanonical depth before bounded deterministic traversal; no polling loop is introduced. |
| T-03-SC | Tampering | package installs | low | accept | No package operation; use existing Decimal, Pytest, and Hypothesis tooling. |
</threat_model>
