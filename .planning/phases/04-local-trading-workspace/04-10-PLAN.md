<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| MainWindow/panel → worker/façade | UI orchestration must not gain trading authority or block event loop。 |
| worker result → current workspace | delayed target-specific result can arrive after switch/close。 |
| approval/kill UI → durable services | confirmations request services but cannot replace their authority。 |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-04-01 | Tampering | stale callback spoofing | high | mitigate | all results carry generation/digest; every slot checks both + UI liveness; switch/close success/error e2e tests are blocking。 |
| T-04-02 | Elevation of Privilege | direct-submit bypass | high | mitigate | MainWindow/panel/dialog constructors never receive gateway/ledger/permit; façade retains strict reader and consume → lease → coordinator path; source behavior tests cover no alternate UI route。 |
| T-04-03 | Information Disclosure | raw exception UI | high | mitigate | only worker redacted safe DTOs render; test controlled error/long-text paths without raw traceback/payload。 |
| T-04-04 | Tampering | false local recovery/terminal state | high | mitigate | status band and dialogs re-render persisted projection, cancellation is non-terminal, stale callbacks are discarded; integration/reopen regressions cover durable truth。 |
| T-04-05 | Denial of Service | GUI-thread blocking | high | mitigate | action handlers only enqueue QThread request; no production timed wait; heartbeat/close tests make freeze a release blocker。 |
</threat_model>
