# Phase 03 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Boundary

**No UI is delivered in Phase 03.** This phase is backend-only: it creates an independently persisted, deterministic PaperGateway and central-ledger projections. It must not add PyQt widgets, screens, scenario controls, navigation, or direct user interactions.

Future operational UI (Phase 4 or later) may only project persisted paper/ledger state and invoke application services. It must never infer order state from local UI state, call a gateway directly, mint approval/permit/lease authority, or advance lifecycle state without committed gateway evidence. This boundary is sourced from `03-RESEARCH.md` (Architectural Responsibility Map) and `ROADMAP.md` (Phase 4 workspace scope).

The remaining sections are **future-projection defaults**, recorded now so later operational surfaces display the durable Paper state consistently. They are not implementation work for this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F7F8FA | Future workspace background and primary reading surfaces |
| Secondary (30%) | #E9EDF2 | Future panel backgrounds, table headers, navigation, and inactive state containers |
| Accent (10%) | #1769AA | Paper-mode badge, selected target, approved/read-only lifecycle evidence, and the primary application-service command only |
| Destructive | #B3261E | Kill-switch state/action, rejection, liquidation, and confirmed cancellation only |

Accent reserved for: the persisted Paper mode indicator; selected product/target; approved or otherwise authoritative lifecycle evidence; and the one primary application-service command in a future operational view. It is never used to imply that an unprojected UI value is authoritative.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Advance Market Scenario |
| Empty state heading | No paper account state is available |
| Empty state body | Persist an initial paper account and market observation through the application service, then reopen this view to inspect the recorded state. |
| Error state | Paper state could not be projected from persisted gateway evidence. Reconcile by client order ID; do not retry submission from this view. |
| Destructive confirmation | Cancel paper order: Cancel the remaining open quantity? Filled quantities and recorded fees will remain unchanged. |

These labels are reserved for a future operational projection surface. Phase 03 does not render them. Copy reflects the sourced lifecycle rules: explicit market observations advance paper state; cancellation is a request resolved by durable evidence; uncertain acceptance requires reconciliation rather than re-submission (`03-CONTEXT.md` D-03, D-09, D-12).
