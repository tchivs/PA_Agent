<interfaces>
`SubmissionCoordinator.submit()` already type-checks `OutboundDispatchPermit`, delegates to `ExecutionLedger.lease_outbound_submission()`, and only then invokes `TradingGateway.submit_order()`. Preserve that order and the existing post-lease ambiguity behavior.

The obsolete `ExecutionLedger.create_or_load_and_claim_submission(ExecutionCommand)` and `begin_outbound_submission(SubmissionAdmission)` currently live in both the protocol and SQLite implementation. Remove them as public production admission paths rather than wrapping or forwarding them. Test setup that needs a submitted/unresolved command must create it through the established ticket-consumption then permit-lease route, or use a test-only persisted fixture that has no production export and cannot return a gateway-facing authorization.

`OutboundSubmission` remains the gateway port input, but it is not a coordinator input. A manually instantiated value must be rejected by the public coordinator boundary before any lease, gateway action, or durable authority mutation.
</interfaces>
