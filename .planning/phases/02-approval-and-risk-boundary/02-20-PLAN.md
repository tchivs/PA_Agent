<interfaces>
`MaskingFormatter.format(record)` currently calls `super().format(record)` before `output_redactor().redact(message)`, so `LogRecord.getMessage()` has already rendered mappings and exceptions into text. A shallow `LogRecord` copy can also inherit `exc_text` from `record.__dict__` after another handler's ordinary formatter has formatted the shared record. `SecretRedactor.redact()` correctly handles Mapping values before serialization, but `_redact_string()` currently masks only registered exact values and URL-query forms.

The fix must preserve standard logging behavior and must not mutate the shared input `LogRecord` seen by other handlers. Build and format a safe record copy: recursively redact `msg`/`args` before `getMessage()` interpolation; the copied record must explicitly clear inherited `exc_text`, or replace it only with a newly constructed safe representation. For every normalized `exc_info` shape (`True`, an exception instance, and an explicit `(type, value, traceback)` tuple), detach the original exception from the copy before calling the standard formatter: emit only its exception type, a controlled reason code, `REDACTION_TOKEN`, and traceback structure that cannot include an exception message. The original exception's string representation, arguments, traceback message, and pre-populated raw `exc_text` must never enter `logging.Formatter`. Extend the centralized string sanitizer for common repr/dict/key-value sensitive shapes so direct string messages are safe too. Do not add logging-specific duplicate masking rules outside `SecretRedactor`.
</interfaces>
</context>
