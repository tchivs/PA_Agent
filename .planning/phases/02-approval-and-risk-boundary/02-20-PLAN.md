<interfaces>
`MaskingFormatter.format(record)` currently calls `super().format(record)` before `output_redactor().redact(message)`, so `LogRecord.getMessage()` has already rendered mappings and exceptions into text. `SecretRedactor.redact()` correctly handles Mapping values before serialization, but `_redact_string()` currently masks only registered exact values and URL-query forms.

The fix must preserve standard logging behavior and must not mutate the shared input `LogRecord` seen by other handlers. Build and format a safe record copy: recursively redact `msg`/`args` before `getMessage()` interpolation, and substitute redacted exception text before standard formatter exception rendering. Extend the centralized string sanitizer for common repr/dict/key-value sensitive shapes so direct string messages are safe too. Do not add logging-specific duplicate masking rules outside `SecretRedactor`.
</interfaces>
</context>
