<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| editable draft → persisted settings | 不可信编辑输入必须先经 typed validation 与安全 policy 比较。 |
| applied config/readiness → approval availability | UI 可见状态不能取代 target/freshness/latch 服务事实。 |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-04-01 | Tampering | draft/validation callback | high | mitigate | draft revision/target digest 纳入 immutable readiness；后续 worker/e2e tests拒绝过期结果写 applied state。 |
| T-04-02 | Elevation of Privilege | configuration policy | high | mitigate | only-tightening baseline comparison 和 façade-owned readiness，widget 无 capability/risk/submit authority。 |
| T-04-03 | Information Disclosure | generic settings/error | high | mitigate | `extra=forbid`、opaque credential reference、safe issue DTO；unit tests拒绝秘密形状输入。 |
| T-04-04 | Tampering | local readiness/recovery claim | high | mitigate | readiness 消费 persisted target/reconciliation/latch prerequisites；不能用 config flag 声称 terminal/READY。 |
| T-04-05 | Denial of Service | validation/save | high | mitigate | 后续 worker 将 validation/save 放入 QThread；本 façade 不触碰 QWidget，pytest-qt tests覆盖 delayed path。 |
</threat_model>
