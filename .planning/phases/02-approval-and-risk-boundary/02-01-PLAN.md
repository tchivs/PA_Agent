<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted analysis record -> trading intent | Untrusted/free-form analysis data is narrowed to immutable canonical execution facts. |
| advisory paths -> trading application | Alerts, notifications, UI callbacks, and LLM output must not acquire a submission capability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering/Elevation | `IntentFactory` | high | mitigate | Strict completed-snapshot schema, reason-coded conversion rejects, and no gateway/ledger submission imports implement D-01 through D-04. |
| T-02-02 | Repudiation | source provenance | medium | mitigate | Persist-ready source ID, completion timestamp, schema version, and deterministic digest are carried by every candidate. |
| T-02-SC | Tampering | package installs | low | accept | No package-manager installation is planned; existing standard-library and project dependencies are used. |
</threat_model>
